# План тестирования для REST API GET: /products/{productId}/status

## Общая информация
Метод: GET  
Эндпоинт: /products/{productId}/status
Авторизация: `authToken` (16-символов строка, состоит из букв и цифр)  
Параметры: `recalculate`, `owner`, `region` (необязательные)
Ожидаемые ответы: 
- HTTP 200 с телом {"productStatus": 1} или {"productStatus": 0}
- HTTP 500 с телом {"errorMessage":}
- HTTP 404

**Примечание:**
1. Текст ошибок условный и зависит от конкретной реализации;
2. Если есть все параметры, то пишем {"productStatus": 1}, иначе - {"productStatus": 0}

# Сводная таблица тест-кейсов

## Позитивные кейсы

| ID | Название | Описание | Ожидаемый результат |
|----|----------|----------|---------------------|
| TC001 | Валидный запрос (все параметры) | Все параметры указаны корректно | HTTP: 200 + JSON { "productStatus": 1 }|
| TC002 | Валидный запрос с одним параметром	|	Все параметры отсутствуют кроме authToken	| HTTP: 200 + JSON { "productStatus": 0 } |

## Негативные кейсы

| ID | Название | Описание | Ожидаемый результат |
|----|----------|----------|---------------------|
| TC003 | Невалидный authToken	|	Значение параметра не набор из 16 символов, включающие цифры и буквы	| HTTP: 500 + JSON { "errorMessage": } |
| TC004 | Невалидный запрос при отсутствии authToken	|	Все параметры указаны корректно, authToken не указан 	| HTTP: 500 + JSON { "errorMessage": "Нет ввода обязательного параметра authToken"} |

| TC006 | Невалидные параметры	|	Один параметр указан некорректно 	| HTTP 500 + JSON { "errorMessage": } |
