# TC006 — Проверка корректности статус-кодов при различных комбинациях recalculate, owner, region

**Предварительные шаги**:
1. Создать тестовый продукт с ID 1234 или убедиться, что он существует в системе.

**Шаги**:
1. Отправить GET-запрос на `/products/1234/status` с параметрами:
   - `authToken=testEqual16simvl`
   - тестовые данные(комбинации параметров):

|  № | recalculate | owner | region | 
|----|----------|----------|---------|
| 1 | true | null| Поволжье | 
| 2 | false | Создатель | null | 
| 3 | null | Пользователь| Северо-Запад |

**Ожидаемый результат**:
- HTTP 200
- Тело ответа: { "productStatus": 1 } или { "productStatus": 0 } (в зависимости от данных в системе)

**Серьезность:** 
Medium.
