# TC003: Обработка некорректного authToken (параметризированный)

**Предварительные шаги**:
1. Создать тестовый продукт с ID 1234 или убедиться, что он существует в системе.

**Шаги**:
1. Отправить GET-запрос для каждого значения authToken на `/products/1234/status` с параметром:
   - Параметры `authToken=`:
   - 
|  № | Вводимое значение | Описание |
|----|------------------|---------------------|
| 1 | Less16simvol | длина меньше 16 символов |
| 2 | testMore16simvols |  длина больше 16 символов |
| 3 | testNoHaveSimvol | не включает в себя цифры |
| 4 | 1234567891012345 | не включает в себя буквы |
| 5 | HaveNot1No)Let__ | содержит специмволы |
| 6 | 123456789.123456 | содержит числа с плавающей точкой |
| 7 | expiredToken1234 | истекший токен |
| 8 | кириллицадоб1234 | используется кириллица |

**Ожидаемый результат**:
- HTTP: 401 или 403
- Тело ответа: `{ "errorMessage": "Недействительный или истекший authToken" }`.

**Серьезность:** 
High
