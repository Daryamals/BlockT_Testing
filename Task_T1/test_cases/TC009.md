# TC009: Обработка внутренней ошибки сервера

**Предварительные шаги**:
1. Создать тестовый продукт с ID 1234 или убедиться, что он существует в системе.
2. Тестовая среда настроена, API доступно, но сервер настроен на возврат внутренней ошибки (например, сбой базы данных).

**Шаги**:
1. Отправить GET-запрос на `/products/1234/status` с валидными параметрами:
   - `authToken=testEqual16simvl` - пользователь не имеет права доступа к нему

**Ожидаемый результат**:
- HTTP: 500
- Тело ответа: `{ "errorMessage": "Внутренняя ошибка сервера" }`.

**Серьезность:** 
Critical
